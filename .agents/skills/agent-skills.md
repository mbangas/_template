# Agent Skills

**Production-grade engineering skills for AI coding agents.**

Skills encode the workflows, quality gates, and best practices that senior engineers use when building software. These ones are packaged so AI agents follow them consistently across every phase of development.


## Skills Catalog

| Skill | Description |
|---|---|
| agile-build | Provides agile build capabilities, allowing for efficient and flexible software development processes. |
| agile-refinement | Performs agile refinement by breaking down user stories into smaller, more manageable tasks. |
| agile-review | Assures documentation and tracking of user stories, tasks, and progress throughout the development lifecycle. |
| doc-api | Analyzes API endpoint code and generates comprehensive OpenAPI specifications for effective API design and communication. |
| doc-create-readme-file | Creates a comprehensive README.md file that effectively communicates the purpose, usage, and contribution guidelines of the project. |
| doc-onboarding | Creates onboarding documentation to help new developers understand the project, set up their environment, and contribute effectively. |
| doc-solution-architecture | Analyzes requirements and generates comprehensive system architecture diagrams and descriptions for effective communication and implementation. |
| rules-api-and-interface-design | Guides stable API and interface design for REST/GraphQL endpoints, module boundaries, and public interfaces. |
| rules-browser-testing-with-devtools | Tests in real browsers by inspecting the DOM, capturing console errors, analyzing network requests, and profiling performance via Chrome DevTools MCP. |
| rules-debugging-and-error-recovery | Guides systematic root-cause debugging when tests fail, builds break, or behavior doesn't match expectations. |
| rules-frontend-ui-engineering | Builds production-quality UIs when creating components, implementing layouts, managing state, or modifying user-facing interfaces. |
| rules-generate-unit-tests | Generates focused unit tests for selected functions or methods, ensuring thorough validation of their behavior. |
| sdlc-code-review | Conducts a thorough code review in the SDLC covering security, performance, quality, architecture, and testing. |
| sdlc-design | Designs software architecture and components based on requirements and plans. |
| sdlc-implement | Implements code based on requirements, plans, designs, and tasks. |
| sdlc-plan | Plans the SDLC for a given requirement by defining scope, objectives, and implementation approach. |
| sdlc-shipping-and-launch | Prepares production launches with pre-launch checklists, monitoring setup, staged rollouts, and rollback strategies. |
| sdlc-tasks | Generates atomic development tasks based on requirements, plans, and designs. |
| sdlc-test | Creates unit tests and tests code based on requirements, plans, designs, and tasks. |


## AGILE Methodology


| Agile Skills    | SDLC Skills                                                                      | What it Does |
|-----------------|----------------------------------------------------------------------------------|--------------|
| agile-refinement | sdlc-plan -> sdlc-design -> sdlc-tasks                                           | Decompose specs into small, verifiable tasks with acceptance criteria and dependency ordering          |
| agile-build     | sdlc-implement -> sdlc-test -> sdlc-code-review                                  | Write the code, Verify and Review Code  |
| agile-review    | doc-solution-architecture -> doc-api -> doc-onboarding -> doc-create-readme-file | Assure Documentation          |
